Description of the illustration cc12102_add_perf_policy.png

The screenshot shows the Add Policy page. The first field on the page is Policy Name, currently empty. The next field is Policy Description, also empty.

The next section on the page is titled Performance Definition for Policy. The descriptive text states "Specify the business ranking and performance objectives for the Performance Classes below. By checking "Measure Only" a Performance Class will be monitored but not managed by QoS Management." In this section you can alter the settings for the rank, Objective value (sec) and Measure-only for each Performance Class.

Below the list of performance classes, there is a section titled "Authorized Actions". The descriptive text states "Select types of resource allocation actions that may be automatically implemented by QoS Management." The available check boxes in this section are:

  • Promote or Demote a Performance Class Consumer Group.

  • Move a CPU between databases within a server pool.

  • Move a server between server pools.

The last section shown in the screenshot is the Server Pool Override section. The descriptive text for this section states "Server Pool Directive Overrides change the server pool availability when the associated policy is in effect." This is followed by an information icon that you can click to obtain more information on the Online Help system. The Server Pool Directive Override section displays the current properties of the server pools and allows you to change the following properties for each server pool:

  • Minimum override value

  • Maximum override value

  • Importance override value

End of description.
