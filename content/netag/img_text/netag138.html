Description of the illustration netag138.eps

The illustration describes the hexadecimal IPv6 format divided into six segments, reading from left to right, where each segment corresponds to a range of bits that represents various parts of an IPv6 address. Starting from the left, segment 1 is 3 bits. Segment 2 is 13 bits and contains TLA ID. Segment 3 is 8 bits and contains RES. Segment 4 is 24 bits and contains NLA ID. Segment 5 is 16 bits and contains Subnet ID. Segment 6 is 64 bits and contains Interface ID.
