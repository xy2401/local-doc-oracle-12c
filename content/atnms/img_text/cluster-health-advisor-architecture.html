This illustration shows the ochad daemon process, with two components inside: the Node Health Prognostics Engine and the Database Health Prognostics Engine. Both of these components of ochad point to the Grid Infrastructure Management Repository (GIMR). The Database Health Prognostics Engine component of ochad gets input from Database instance (DB) data, and the Node Health Prognostics Engine component of ochad gets input from operating system data, which comes from Cluster Health Monitor (CHM).
